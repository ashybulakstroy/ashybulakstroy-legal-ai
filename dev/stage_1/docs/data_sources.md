# Схема данных: adilet.zan.kz vs kodeksy-kz.com

## Источники

### 1. Adilet (adilet.zan.kz) — официальный источник
- Официальная база НПА Республики Казахстан
- **14 законов**, **~5181 статья** в текущей БД
- Полные метаданные: номер закона, дата принятия, статус, полный текст (`full_text`)
- Структурированные статьи с номерами и заголовками
- Статус определяется из баннера сайта ("Утративший силу", "Обновленный")

**Парсер**: `AdiletParser` (синхронный, httpx)
- Извлекает: title, number, date_adopted, status, summary, full_text, articles
- Статьи парсятся из `<div class="module_npaView">` по заголовкам "Статья X."
- Конституция парсится отдельно (нет номера, но есть дата)

### 2. Kodeksy (kodeksy-kz.com) — неофициальный источник
- Неофициальный сайт с текстами законов РК
- **20 законов**, **~2219 статей** в текущей БД
- Нет метаданных: номер закона, дата принятия, полный текст — всё `NULL`
- Статьи загружаются постранично (1 URL = 1 статья)
- Статус определяется по ключевым словам на странице

**Парсер**: `KodeksyParser` (асинхронный, httpx)
- Извлекает: title, status, summary (= первая статья), articles
- Не извлекает: number, date_adopted, full_text
- Содержал много boilerplate (удалён скриптом `clean_articles.py`):
  - Заголовки "Ст. N Закон..." в начале статей
  - Строки статуса и версии
  - Примечания РЦПИ

## Дубликаты

**Дубликатов между источниками нет** — каждый закон загружен ровно из одного источника.
Единственная группа с похожими названиями — две части ГК (Общая и Особенная), что корректно.

## Оценка необходимости дедупликации

### Текущее состояние
- Дубликаты **отсутствуют** — дедупликация не требуется
- Если в будущем закон загружается из обоих источников — дубликат по title, number + date_adopted

### Предлагаемая стратегия дедупликации
1. **Первичный ключ**: `(title[:50], number)` — если number не NULL
2. **Без номера**: сравнение по совпадению заголовков >80% (через difflib)
3. **При конфликте**: adilet имеет приоритет (официальный источник с метаданными)

## Пробелы и рекомендации

### Основные пробелы
1. **14 из 19 законов РК из неофициального kodeksy** — без номера и даты принятия
2. **УИК, Социальный, Земельный кодексы** — тоже из kodeksy
3. **Нет полного текста (full_text) для 20 законов**
4. **Много законов РК вообще не загружено** — adilet.zan.kz содержит сотни законов
5. **Нет дат date_effective и date_expired** ни для одного закона

### Рекомендации
- **Приоритет**: перезагрузить kodeksy-законы с adilet (официальный источник с полными метаданными)
- Для массовой загрузки с adilet нужно изучить их API/поиск (сейчас load_laws.py использует hardcoded doc_id)
- Альтернатива: оставить kodeksy как fallback, но добавить source_url в модель Law
- Добавить `source` и `source_url` поля в Law для прозрачности происхождения данных
- date_effective можно заполнять из adilet (часто указан в мета-тексте)
- date_expired — только для repealed законов, редко встречается
